# Eval Report Metrics

This folder stores JSON reports generated by:

`src/realtime/realtime_inference.py --report <path>`

## Report Structure

Each report contains run metadata, performance metrics, stability metrics, and threshold results.

## Field Reference

| Field | Type | Meaning |
|---|---|---|
| `mode` | string | `file` (offline chunked replay) or `live` (real microphone/speaker stream). |
| `timestamp` | string (ISO-8601) | UTC time when report was created. |
| `host_id` | string | Hostname of the machine running evaluation. |
| `device` | string | Torch device used (`cpu`, `mps`, `cuda`, etc.). |
| `sample_rate` | int | Audio sample rate in Hz. |
| `chunk_size` | int | Processing chunk size in samples. |
| `stft_pad_size_samples` | int | STFT lookahead/pad size in samples. |
| `chunks` | int | Number of chunks processed in the run. |
| `warmup_chunks_excluded` | int | Number of initial chunks excluded from most aggregate metrics. |
| `rtf_avg` | float | Average real-time factor. Computed as `mean(chunk_latency_ms) / chunk_duration_ms`. Lower is better; `< 1.0` means faster than real-time. |
| `latency_ms_avg` | float | Mean per-chunk processing latency (ms), post-warmup. |
| `latency_ms_p50` | float | 50th percentile latency (ms), post-warmup. |
| `latency_ms_p95` | float | 95th percentile latency (ms), post-warmup. |
| `latency_ms_p99` | float | 99th percentile latency (ms), post-warmup. |
| `latency_ms_max` | float | Max latency (ms), post-warmup. |
| `drops_input` | int | Number of input chunks dropped because input queue was full. Live mode metric. |
| `drops_output` | int | Number of old output chunks dropped to control latency buildup. Live mode metric. |
| `underruns` | int | Number of times output callback had no data and played silence. Live mode metric. |
| `nan_count` | int | Count of NaN output samples. This is counted from chunk 0 (including warmup). |
| `clip_ratio` | float | Fraction of output samples at clipping boundary (`abs(sample) >= 1.0`), post-warmup. |
| `rms_in` | float | RMS (Root Mean Square) level of input audio, post-warmup. RMS is computed as `sqrt(mean(x^2))` over samples and is a power/loudness proxy. |
| `rms_out` | float | RMS (Root Mean Square) level of output audio, post-warmup. RMS is computed as `sqrt(mean(x^2))` over samples and is a power/loudness proxy. |
| `status` | string | `pass` or `fail` after threshold evaluation. |
| `failed_thresholds` | string[] | Metric names that violated the selected threshold profile. |

## Notes

- In `file` mode, `drops_input`, `drops_output`, and `underruns` are stubbed to `0` for schema consistency.
- Threshold checks for those three stability metrics are excluded in `file` mode.
- If no `--threshold-profile` is provided, `failed_thresholds` is empty and `status` is `pass`.

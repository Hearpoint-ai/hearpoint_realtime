# Eval Reports

JSON reports are generated by `make eval-fast` or `make demo` (when `test.enabled: true` in `config.yaml`).

## Output Files

All outputs share a timestamp prefix (`YYYYMMDDTHHmmss`) generated at run start:

| Path | Contents |
|---|---|
| `reports/eval/<ts>.json` | Metrics report |
| `reports/eval/plots/<ts>_spectrogram.png` | Spectrogram comparison (mixture / output / reference) |
| `reports/eval/plots/<ts>_rtf_distribution.png` | Per-chunk RTF histogram |
| `reports/eval/plots/<ts>_sisdri_vs_snr.png` | SI-SDRi vs input SNR scatter (1-second segments) |
| `reports/eval/output_audio/<ts>.wav` | Enhanced output audio |

## Field Reference

| Field | Type | Meaning |
|---|---|---|
| `mode` | string | `file` (chunked replay) or `live` (real mic/speaker stream). |
| `timestamp` | string | UTC time when report was created. |
| `host_id` | string | Hostname of the machine running evaluation. |
| `device` | string | Torch device used (`cpu`, `mps`, `cuda`, `coreml`). |
| `sample_rate` | int | Audio sample rate in Hz. |
| `chunk_size` | int | Processing chunk size in samples. |
| `stft_pad_size_samples` | int | STFT lookahead/pad size in samples. |
| `chunks` | int | Total chunks processed. |
| `warmup_chunks_excluded` | int | Initial chunks excluded from aggregate metrics. |
| `rtf_avg` | float | Average real-time factor (`mean latency / chunk duration`). `< 1.0` = faster than real-time. |
| `latency_ms_avg` | float | Mean per-chunk processing latency (ms), post-warmup. |
| `latency_ms_p50/p95/p99` | float | Latency percentiles (ms), post-warmup. |
| `latency_ms_max` | float | Max latency (ms), post-warmup. |
| `drops_input` | int | Input chunks dropped (queue full). Live mode only. |
| `drops_output` | int | Output chunks dropped to control latency buildup. Live mode only. |
| `underruns` | int | Output callback played silence (no data ready). Live mode only. |
| `nan_count` | int | NaN output samples (counted from chunk 0, including warmup). |
| `clip_ratio` | float | Fraction of output samples at clipping boundary, post-warmup. |
| `rms_in` / `rms_out` | float | RMS level of input / output audio, post-warmup. |
| `cosine_similarity_before` | float | Speaker cosine similarity of mixture to enrollment. |
| `cosine_similarity_after` | float | Speaker cosine similarity of enhanced output to enrollment. |
| `cosine_similarity_delta` | float | Improvement in cosine similarity (`after - before`). |
| `si_sdr_input` | float | SI-SDR (dB) of mixture vs reference. |
| `si_sdr_output` | float | SI-SDR (dB) of enhanced output vs reference. |
| `si_sdr_improvement` | float | SI-SDRi (dB): `si_sdr_output - si_sdr_input`. |
| `status` | string | `pass` or `fail` after threshold evaluation. |
| `failed_thresholds` | string[] | Metric names that violated the threshold profile. |

## Notes

- In `file` mode, `drops_input`, `drops_output`, and `underruns` are `0` (stubbed for schema consistency) and excluded from threshold checks.
- Cosine metrics require a `<embedding>.meta.json` sidecar alongside the `.npy`; use `media/si_sdr_fixture/enrollment.npy` or re-enroll to generate one.
- If no `threshold_profile` is set, `failed_thresholds` is empty and `status` is always `pass`.
